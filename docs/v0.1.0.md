# v0.1.0 更新记录

发布日期：2026-04-11

## 功能模块

### 音频播放器
- 浏览器端音频队列播放
- POST /api/play 入队，自动依次播放
- 跳过（/api/skip）、清空（/api/clear）
- WebSocket /ws 实时推送 play/queued/cleared 事件
- 配合 ComfyUI + CosyVoice 语音合成使用

### 英语学习模块（Vocab）
- 粘贴英文文章 → 自动分词、统计词频、入库
- 单词状态着色：红色(unknown) / 黄色(familiar) / 白色(mastered)
- 点击单词切换状态：unknown/mastered → familiar → mastered
- 左侧栏词汇统计和文章历史
- API：文章 CRUD、单词状态更新、词汇统计

### 数据模型
- zVocab：单词表（word, status, first_seen, last_seen, click_count）
- zArticles：文章表（title, content, created）
- zWordArticles：单词-文章关联表（word, article_id, count）

## 部署方式

### Docker 容器化
- 镜像：zdashboard:latest，基于 node:20-alpine
- 配置通过环境变量：HOST, DB_PATH, AUDIO_DIR
- docker-compose 一键启动

### .run 便携部署
- `bash deploy/build_run.sh` 构建
- 生成自解压可执行文件 zdashboard.run
- 环境变量：ZDB_PATH, ZAUDIO_DIR, ZDASHBOARD_PORT（无默认值，必须 export）
- 每次运行强制 docker load（避免旧镜像缓存）
- 自动 kill 旧容器并重启

## 修复与优化记录

- **分词回滚**：从 NLP 分词回滚为正则匹配（含数字），修复文本选择和布局跳动
- **CSS/JS 拆分**：单文件 index.html 拆分为 index.html + style.css + app.js
- **Vocab/Article bug 修复**：重构时引入的路由和数据问题
- **Dockerfile 修复**：从 git clone 改为 COPY（容器内无代理），添加 python3/make/g++ 编译 better-sqlite3
- **配置分离**：数据库路径、端口等从硬编码改为环境变量 > config.json > 默认值
- **Host 配置**：HOST 加入配置，Docker 部署时可设为 0.0.0.0
- **zdb 挂载**：从 Docker named volume 改为直接挂载权威 zdb.db 文件
- **.run 旧镜像缓存**：startup.sh 每次强制 docker load，避免部署新 .run 仍用旧镜像
- **.run 旧容器处理**：startup.sh 增加 docker stop + docker rm，确保重启时清理旧容器

## 技术栈

- 后端：Node.js (Express + ws + better-sqlite3)
- 前端：单文件 SPA，深色主题，横竖屏自适应
- 数据库：SQLite (zdb.db)
- 部署：Docker + makeself .run

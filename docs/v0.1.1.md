# v0.1.1 更新记录

发布日期：2026-04-11

## 部署流程重构

### c2r 集成
- 构建脚本从手写 docker save + makeself 迁移为调用 [container2run (c2r)](https://github.com/HanqingTony/container2run)
- `deploy/pack/` 目录已删除（startup.sh 由 c2r 自动生成）
- `docker-compose.yml` 从 `deploy/pack/` 提升到 `deploy/`（c2r & compose 共用）
- `build_run.sh` 代码量从 165 行缩减到 29 行（-82%）

### 版本化构建
- 镜像不再使用 `latest` 标签，改为从 `package.json` 读取语义化版本号
- 默认构建为 `zdashboard:x.y.z`，同时打 `latest` 便利标签
- 导出的 tar 文件名带版本号：`zdashboard-x.y.z.tar.gz`
- 可手动指定版本：`bash deploy/build_run.sh 1.2.0`

### 构建产物全保留
`zbuild/` 目录现在保留所有构建产物：

```
zbuild/
├── zdashboard.run              ← c2r 打包的可执行部署文件
├── zdashboard-0.1.1.tar.gz     ← 版本化的镜像 tar（可手动 docker load）
├── Dockerfile                  ← 构建副本
└── docker-compose.yml          ← 配置副本
```

## 修复与优化

- **c2r bug 修复**（贡献给 container2run 项目）：
  - capture 提取端口用 `HostConfig.PortBindings` 替代 `NetworkSettings.Ports`（后者对停止的容器返回空）
  - Windows CRLF compose 文件导致镜像名带 `\r`，docker 命令报 invalid reference format
  - `detect_image_origin` 的 log 输出污染 `$()` 返回值，导致 embed 逻辑失效
  - `docker save | gzip` 管道静默吞错误，缺少预检
  - `latest` 标签策略：mutable tag 不保护，退出时一律删除（零残留）
